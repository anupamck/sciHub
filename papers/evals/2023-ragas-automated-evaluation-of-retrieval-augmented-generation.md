## Why I saved this

- Core paper for evaluating RAG systems systematically beyond ad-hoc manual checks.

## Key points

- Proposes automated metrics for critical RAG dimensions without requiring gold reference answers.
- Separates retrieval and generation quality signals, making failures easier to diagnose.
- RAGAS outperformed crude ChatGPT evaluation on all three metrics.

## Methodology

- 50 Wikipedia pages on post-2022 events (beyond the model's training cutoff).
- Questions and answers generated by ChatGPT, then annotated by 2 human annotators.
- For each metric, a synthetically degraded low-quality candidate was paired with the standard answer; annotators and RAGAS both picked the better one.
- Agreement measured as pairwise accuracy against human labels.

## Notes

- Useful baseline before designing custom evaluation rubrics for domain-specific RAG use cases.


## Limitations

- Was tested with merely 2 human annotators. 
- **WikiEval is tiny (50 instances)**: The entire validation rests on 50 question-context-answer triples. This is far too small to make strong statistical claims about agreement with human judgment, and confidence intervals are never reported.
- **Pairwise contrast is artificially coarse**: The low-quality candidates are synthetically degraded (context-free answers for faithfulness, explicitly incomplete answers for answer relevance, padded contexts for context relevance). Real-world failure modes are far subtler — the setup likely overstates how well RAGAS discriminates in practice.
- **Answer relevance only tests incompleteness**: The metric claims to penalize both incomplete and redundant answers, but the validation dataset only constructs incomplete low-quality candidates, leaving the redundancy half unvalidated.
- **Self-referential evaluation**: Both the dataset construction and the metric computation use the same underlying model (ChatGPT / gpt-3.5-turbo). The high faithfulness score (0.95) may partly reflect the model being consistent with itself rather than genuinely aligning with human judgment.
- **Context relevance is the weakest metric**: Accuracy of 0.70 is only modestly above the 0.63 of the simplest baseline. The extraction prompt used inside the metric is the same mechanism that struggles — so the metric's ceiling is limited by the model's own weakness at long-context sentence selection.
- **No domain diversity**: All 50 Wikipedia pages cover recent world events. Performance on technical, legal, medical, or scientific RAG pipelines — where faithfulness evaluation is most critical — is untested.
- **No discussion of prompt sensitivity**: Results for all three methods are reported for a single prompt design. The paper acknowledges that LLM-based scoring is sensitive to prompt wording, but no ablation is run.
